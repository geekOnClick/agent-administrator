# SOP: process_bills

## Назначение
`process_bills` обрабатывает счета (`.xlsx`, `.xls`, `.pdf`, `.doc`, `.docx`), извлекает суммы, считает общий итог и сохраняет текстовый отчет.

## Входные параметры
- `paths` (обязательно): массив путей к файлам и/или папкам со счетами.
- `outputPath` (опционально): путь для сохранения отчета. Если не указан, отчет создается рядом с первым найденным счетом.

## Шаги выполнения
1. Принять и провалидировать параметры по JSON-схеме.
2. Развернуть список `paths` до списка допустимых файлов-счетов.
3. Для каждого файла:
   - прочитать содержимое;
   - извлечь сумму к оплате;
   - добавить запись в детализацию.
4. Суммировать найденные значения и вычислить итог.
5. Сформировать текстовый отчет и записать его в `outputPath` (или автосгенерированный путь).
6. Вернуть:
   - текстовый итог для пользователя;
   - `structuredContent` с `reportPath`, `total`, `entries`.

## Формат успешного результата
- `content[0].text`:
  - итоговая сумма;
  - путь к отчету;
  - детализация по каждому файлу.
- `structuredContent`:
  - `reportPath: string`
  - `total: number`
  - `entries: Array<{ file: string; amount: number }>`

## Ошибки
При исключении возвращается:
- `isError: true`
- текст `Ошибка обработки счетов: ...`

## Операционные требования
- Использовать только поддерживаемые форматы документов.
- Проверять существование файлов и директорий до обработки.
- Не удалять исходные документы.
- Логировать только техническую информацию без секретов.
